# Semantic Types

## Semantic Types

Semantic types provide a way to attach additional meaning and validation rules to data. They are used to define the expected structure and content of various data elements within the system.

### Core Semantic Types

#### `str`
Represents a string value. Can be used with semantic string literals.
```jac
str "This is a semantic string."
```

#### `int`
Represents an integer value.

#### `float`
Represents a floating-point number.

#### `bool`
Represents a boolean value (`True` or `False`).

#### `list`
Represents an ordered collection of items.
```jac
list[str] my_list = ["item1", "item2"]
```

#### `dict`
Represents a collection of key-value pairs.
```jac
dict[str, int] my_dict = {"key1": 1, "key2": 2}
```

#### `set`
Represents an unordered collection of unique items.
```jac
set[str] my_set = {"apple", "banana"}
```

#### `tuple`
Represents an ordered, immutable collection of items.
```jac
tuple[int, str] my_tuple = (1, "hello")
```

### Special Semantic Types

#### `type`
Used for type annotations, indicating the expected type of a variable or parameter.
```jac
has x: type
```

#### `any`
Represents a value that can be of any type.

#### `null`
Represents the absence of a value.

#### `bytes`
Represents a sequence of bytes.

#### `yaml`
Represents a YAML formatted string.

#### `csv`
Represents a CSV formatted string.

#### `file_path`
Represents a path to a file.

#### `directory_path`
Represents a path to a directory.

#### `regex`
Represents a regular expression pattern.

#### `password`
Represents a sensitive string, typically for passwords.

#### `secret`
Represents a sensitive string, similar to `password`.

#### `enum`
Represents a type whose value must be one of a predefined set of named constants.
```jac
enum Status {
    ACTIVE,
    INACTIVE,
    PENDING
}
```

#### `union`
Represents a value that can be one of several specified types.
```jac
has data: str | int
```

#### `optional`
Represents a value that can be either a specified type or `null`.
```jac
has value: optional[str]
```

#### `literal`
Represents a value that must be exactly one of the specified literal values.
```jac
has color: literal["red", "green", "blue"]
```

#### `callable`
Represents a function or method.
```jac
has callback: callable[[str], int] # A function taking a string and returning an int
```
